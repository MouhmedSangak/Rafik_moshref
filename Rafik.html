🎯 ما هو التطبيق؟

رفيق | مشرف متابعة هو تطبيق ويب محلي مصمم خصيصًا للمشرفين التربويين والمعلمين وأصحاب المراكز التعليمية الذين يتابعون طلابهم عبر واتساب.

يوفر التطبيق واجهة عربية حديثة وسهلة الاستخدام لإدارة بيانات الطلاب، إرسال الرسائل الفردية والجماعية، تتبع التواصل، وتنظيم المتابعة اليومية بشكل احترافي ومنظم.

💡 ما المشكلة التي يحلها؟

يواجه المشرفون التربويون والمعلمون تحديات يومية في:

  • إدارة عدد كبير من الطلاب وبياناتهم بشكل مبعثر (دفاتر، جداول Excel، ملفات متفرقة)
  • صعوبة إرسال الرسائل الجماعية عبر واتساب لعشرات الطلاب يدويًا
  • نسيان متابعة الطلاب الذين لم يتم التواصل معهم منذ فترة
  • عدم وجود سجل منظم للرسائل والمتابعات السابقة
  • استهلاك الوقت والجهد في البحث عن أرقام الطلاب ونسخ الرسائل يدويًا

رفيق يحل هذه المشاكل من خلال توفير نظام متكامل لإدارة العلاقة بين المشرف والطالب عبر واتساب.

✨ أهم المميزات والوظائف

1. إدارة الطلاب 👨‍🎓
  • إضافة، تعديل، وحذف بيانات الطلاب
  • تصنيف الطلاب حسب الحالة (نشط، يحتاج متابعة، غير نشط)
  • إضافة وسوم (Tags) لتنظيم الطلاب
  • استيراد جماعي للطلاب من قائمة أرقام
  • بحث وفلترة سريعة
2. إرسال رسائل واتساب 💬
  • إرسال رسائل فردية مباشرة لطالب معين
  • إرسال رسائل جماعية لمجموعة مختارة من الطلاب
  • قوالب رسائل جاهزة (تشجيع، تذكير، متابعة، تحفيز)
  • معاينة الرسالة قبل الإرسال
  • استخدام متغيرات ديناميكية مثل {student_name}
3. متابعة التواصل 📋
  • تسجيل تاريخ آخر تواصل مع كل طالب
  • عرض الطلاب الذين لم يتم التواصل معهم منذ 7 أيام
  • عرض الطلاب المتأخرين (14 يوم بدون تواصل)
  • سجل كامل للرسائل المرسلة
4. لوحة تحكم شاملة 📊
  • إحصائيات سريعة (عدد الطلاب، الرسائل، المتابعات)
  • نشاطات حديثة
  • قوائم سريعة للمتابعة
5. واجهة عربية جميلة 🌸
  • تصميم عصري وجذاب
  • ألوان مريحة ومنظمة
  • رموز تعبيرية (Emojis) في كل مكان
  • استجابة كاملة على جميع الأجهزة

🎨 تجربة المستخدم داخل التطبيق

عند فتح التطبيق، يظهر شريط جانبي أرجواني جميل يحتوي على:

  • 🏠 الرئيسية
  • 👨‍🎓 الطلاب
  • 💬 المحادثات
  • 📢 إرسال جماعي
  • ✨ القوالب
  • 📋 المتابعة
  • 💚 واتساب
  • 📊 الإحصائيات
  • ⚙️ الإعدادات

رحلة المستخدم النموذجية:

  1. الاتصال بواتساب: المشرف يفتح صفحة واتساب ويمسح QR Code مرة واحدة
  2. إضافة الطلاب: يضيف طلابه يدويًا أو يستورد قائمة أرقام جماعية
  3. إنشاء قوالب: ينشئ قوالب رسائل جاهزة (مثل: "بارك الله فيك يا {student_name}...")
  4. الإرسال الجماعي: يختار مجموعة طلاب، يختار قالب أو يكتب رسالة، ويرسل للجميع
  5. المتابعة اليومية: يفتح صفحة المتابعة ليرى من يحتاج تواصل ويرسل لهم مباشرة

⚡ كيف يسهّل إدارة العمل والمتابعة

بدون رفيق مع رفيق
البحث في دفتر أو Excel عن رقم الطالب البحث الفوري بالاسم أو الرقم
نسخ ولصق الرسالة 40 مرة يدويًا إرسال لـ 40 طالب بضغطة واحدة
نسيان متابعة بعض الطلاب تنبيهات تلقائية للطلاب المتأخرين
ضياع سجلات الرسائل سجل كامل ومنظم لكل رسالة
استهلاك ساعات في العمل اليدوي إنجاز نفس المهمة في دقائق

🔧 القسم الثاني: الجانب التقني

🏗️ البنية التقنية للتطبيق

رفيق مبني على معمارية ويب محلية (Local Web App) تجمع بين:

  • Backend قوي ومستقر
  • Frontend جميل وتفاعلي
  • Automation Engine لإدارة واتساب ويب

💻 نوع المنصة

تطبيق ويب محلي (Local Web Application)

  • يعمل على جهاز المستخدم مباشرة
  • يُفتح في المتصفح عبر http://127.0.0.1:5000
  • لا يحتاج إنترنت لإدارة البيانات (فقط لإرسال الرسائل)
  • لا توجد خوادم خارجية (البيانات محفوظة محليًا فقط)

مزايا هذا النموذج:

✅ خصوصية كاملة

البيانات لا تغادر جهازك أبدًا

✅ لا رسوم اشتراك

لا توجد رسوم شهرية

✅ سريع ومستقر

أداء عالي ومستقر

✅ سهل التثبيت

سهل التثبيت والتشغيل

⚙️ التقنيات المستخدمة

Backend (الخلفية)

  • Python 3.10+ - لغة البرمجة الرئيسية
  • Flask - إطار عمل الويب الخفيف
  • SQLAlchemy - للتعامل مع قاعدة البيانات بشكل احترافي
  • Playwright - لأتمتة واتساب ويب

Frontend (الواجهة)

  • HTML5 - البنية الأساسية للصفحات
  • CSS3 - التصميم والألوان والتنسيق
  • JavaScript (jQuery) - التفاعل الديناميكي
  • Font Awesome - الأيقونات
  • Google Fonts (Cairo) - خط عربي جميل

Automation (الأتمتة)

  • Playwright (Chromium) - فتح واتساب ويب وإرسال الرسائل تلقائيًا
  • Threading - لإدارة العمليات بالتوازي
  • Session Management - حفظ جلسة واتساب

💾 نظام قاعدة البيانات

SQLite - قاعدة بيانات محلية خفيفة وسريعة

هيكل الجداول الرئيسية:

الجدول الوصف
students بيانات الطلاب (الاسم، الرقم، الحالة، الوسوم، آخر تواصل)
messages سجل الرسائل المرسلة (الطالب، المحتوى، التاريخ، الحالة)
templates قوالب الرسائل الجاهزة (الاسم، الفئة، المحتوى)
settings إعدادات التطبيق (الفاصل الزمني، الإعدادات العامة)
activities سجل النشاطات (إضافة طالب، حذف، إرسال)

مسار قاعدة البيانات:

🔐 آلية إدارة الصلاحيات

النسخة الحالية:

التطبيق مصمم لمستخدم واحد فقط (مشرف واحد).

التطوير المستقبلي (إن طُلب):

يمكن إضافة نظام صلاحيات متعدد:

الدور الصلاحيات
المشرف الرئيسي كامل الصلاحيات (إضافة، حذف، إرسال، إعدادات)
المساعد إرسال رسائل وعرض الطلاب فقط
مراقب عرض الإحصائيات والتقارير فقط

🔗 التكاملات المحتملة

✅ موجود حاليًا:

  • WhatsApp Web API (عبر Playwright)
  • حفظ جلسة واتساب محليًا

🚀 قابل للإضافة مستقبلًا:

  • تصدير التقارير PDF/Excel
  • إشعارات سطح المكتب عند استلام رسالة
  • تكامل مع Google Sheets لاستيراد الطلاب
  • ربط مع Telegram Bot للتنبيهات
  • تقارير أسبوعية تلقائية
  • نسخ احتياطي تلقائي للسحابة (Google Drive/Dropbox)

🛡️ مستوى الأمان وحماية البيانات

✅ الأمان المُطبّق:

1. خصوصية كاملة:

  • جميع البيانات محفوظة محليًا فقط على جهاز المستخدم
  • لا توجد خوادم خارجية
  • لا تُرسل أي بيانات لأي جهة

2. حماية جلسة واتساب:

  • الجلسة محفوظة بشكل آمن في مجلد data/whatsapp_session
  • لا يتم تخزين كلمات مرور واتساب

3. عدم استخدام واجهات غير رسمية:

  • التطبيق يستخدم واتساب ويب الرسمي
  • لا توجد واجهات برمجية غير موثوقة

4. نسخ احتياطي يدوي:

  • المستخدم يمكنه نسخ ملف data/rafiq.db في أي وقت

🔒 توصيات إضافية (للإنتاج):

  • تشفير قاعدة البيانات المحلية
  • إضافة كلمة مرور لفتح التطبيق
  • نسخ احتياطي تلقائي مشفر

📦 متطلبات التشغيل

الحد الأدنى:

  • نظام التشغيل: Windows 10+, macOS, Linux
  • Python: 3.8 أو أحدث
  • المتصفح: Chrome/Edge (مثبت تلقائيًا مع Playwright)
  • الذاكرة: 4 GB RAM
  • المساحة: 500 MB مساحة حرة

التثبيت:

pip install -r requirements.txt
playwright install chromium
python main.py

بدء التشغيل:

يفتح التطبيق تلقائيًا في المتصفح على http://127.0.0.1:5000
